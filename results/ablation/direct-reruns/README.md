# Direct Ablation Reruns

This directory stores no-context sub-agent reruns and direct archived-evidence ledgers used to replace derived/proxy ablation rows with direct evidence.

## Arms

- `no-chart-param-layer2/` — `NoChartParamRefsAgent` on the 30 NVBench Layer-2 chart tasks. The sub-agent used task text and schema only, without Superset chart-parameter references. Scoring requires Superset artifact execution, so the row records strong spec coverage but `0/30` verified chart artifacts.
- `no-chart-param-layer3-dashboard/` — `NoChartParamRefsAgent` on the 10 BIRD `financial` dashboard tasks. The sub-agent produced dashboard specs without creating Superset dashboards.
- `no-rca-drill-planning/` — `NoRcaDrillPlanningAgent` on the 10 BIRD `financial` RCA tasks. The sub-agent did not receive the RCA drill-planning SOP; its SQL Lab probe failed with missing authorization, so outputs are scored as planned SQL / inconclusive reports without executed evidence.
- `no-verify-repair/` — direct evidence ledger for the archived first-pass/no-retry SkillGuided outputs. It excludes the BIRD repair-loop outputs and records class scores consumed by the combined ablation aggregator.
- `screenshot-only-monitoring/` — `ScreenshotOnlyMonitoringAgent` direct signal-limited scoring. The scorer saw only task text plus the ten rendered financial dashboard screenshots; it did not read result JSON, score sheets, or ablation summaries.
- `api-only-monitoring/` — `ApiOnlyMonitoringAgent` direct signal-limited scoring. The scorer saw only task text plus structured SkillGuided JSON evidence; it did not inspect PNG screenshots, screenshot manifests, score sheets, or ablation summaries.
- `inputs/` — exact JSON payloads returned by the three sub-agents before conversion to the standard per-task result JSON layout.

## Scoring

- Layer-2 chart scores are generated by `scripts/bench/grade_nvbench_layer2.mjs`.
- Dashboard and RCA scores are generated by `scripts/bench/grade_financial.mjs`.
- Signal-limited monitoring rows are scored from `signal-limited-score.json` files produced by restricted no-context scorer agents.
- The combined table is regenerated by `scripts/bench/aggregate_ablation_table.mjs`.
